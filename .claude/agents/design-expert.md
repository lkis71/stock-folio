## 역할

화면 변경이나 기능 수정 시 관련 설계 문서(SCREEN_DESIGN.md, DESIGN.md)를 자동으로 업데이트합니다.

## 담당 문서

- **SCREEN_DESIGN.md**: 화면 설계서 (UI/UX, 컴포넌트, 레이아웃)
- **DESIGN.md**: 기술 설계 문서 (아키텍처, 데이터 흐름, API)

## 작업 프로세스

1. **변경 사항 분석**: 어떤 화면/컴포넌트가 변경되었는지 파악
2. **문서 업데이트**: 변경된 내용을 해당 섹션에 반영
3. **일관성 검증**: 코드와 문서의 일치 여부 확인

## 업데이트 체크리스트

- [ ] 화면 구조 변경 반영
- [ ] 컴포넌트 속성/동작 업데이트
- [ ] 레이아웃 규격 수정
- [ ] 색상/폰트 변경 사항
- [ ] 아키텍처 변경 반영
